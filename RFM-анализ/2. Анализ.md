## О бизнесе А.Аптека
Аптечная сеть А.Аптека, входящая в экосистему Андекс, является крупным и надежным поставщиком лекарств и медицинских товаров. Компания предлагает широкий ассортимент продукции, начиная от лекарственных препаратов и медицинских изделий до товаров для здоровья и красоты.

Аптеки этой сети отличаются высоким уровнем обслуживания и профессионализмом персонала. Клиенты могут получить квалифицированную консультацию по вопросам здоровья, правильного применения лекарств и выбора необходимых средств.

Важным преимуществом работы с данной аптечной сетью является доступность и удобство: аптеки расположены по всему городу, что обеспечивает удобство для клиентов.

Благодаря вхождению в экосистему Андекс, компания имеет доступ к передовым технологиям и инновационным решениям в области здравоохранения, что позволяет улучшать качество обслуживания и удовлетворять потребности клиентов на самом высоком уровне.

На данный момент существует система почтовых рассылок с акциями и новостями компании. Однако сейчас рассылки писем одинаковы для всех пользователей. В связи с уменьшением спроса в теплое время года руководство компании задумалось о том, чтобы ввести в систему персонализированные письма. Для решения задачи был выбран RFM-анализ. 
## Цели и задачи
**Цель исследования** – с помощью анализа данных получить классификацию пользователей и представить стратегию для работы с email-рассылкой.
Для достижения цели были поставлены следующие **задачи**:
1. Исследовать структуру базы данных компании.
2. Рассчитать метрики распределения величин.
3. Подобрать границы значений метрик.
4. Классифицировать пользователей.
5. Изучить результаты.
6. Дать рекомендации по плану персонализированных писем.
## Об RFM-анализе
RFM-анализ (Recency, Frequency, Monetary) — это метод анализа клиентской базы данных, используемый для сегментации клиентов на основе трех основных параметров:
- **Recency** (Недавность) - оценка времени, прошедшего с момента последней покупки клиента. Чем более недавно клиент совершил покупку, тем выше его оценка по параметру Recency.
- **Frequency** (Частота) - количество покупок, совершенных клиентом за определенный период времени. Клиенты, которые совершают покупки чаще, получают более высокую оценку по параметру Frequency.
- **Monetary** (Денежный объем) - суммарная стоимость всех покупок, совершенных клиентом за определенный период времени. Клиенты, которые приносят больший доход компании, получают более высокую оценку по параметру Monetary.
 
| Грейд | Recency | Frequency | Monetary |
|:----|:----|:----|:----|
| 1 | Давние клиенты | Покупает очень редко (единичные заказы) | Маленькая сумма |
| 2 | Сравнительно недавние клиенты | Покупает нечасто | Средняя сумма |
| 3 | Свежие клиенты | Постоянный покупатель | Большая сумма |

После анализа данных с помощью RFM-метода компании могут проводить следующие действия:
- Создавать индивидуализированные предложения и скидки для категорий клиентов, основанные на их поведении покупательного поведения.
- Разрабатывать стратегии по привлечению новых клиентов, направленные на привлечение покупателей с показателями, близкими к показателям лояльных клиентов.
- Определять потенциальных партнеров для совместной работы или сотрудничества, исходя из анализа покупательской базы.
- Улучшать клиентский опыт и удовлетворенность, ориентируясь на идентифицированные проблемы и нужды различных групп клиентов.

RFM-анализ является важным инструментом для оптимизации маркетинговых и продажных стратегий и повышения эффективности взаимодействия с покупателями.
## Структура базы данных
Для анализа понадобятся не все таблицы базы данных компании, а только данные по покупкам с бонусными картами, чтобы идентифицировать пользователей. Данные находятся в таблице `bonuscheques`. Необходимые поля:

- `datetime` - дата и время заказа
- `card` - номер карты лояльности
- `summ_with_disc` - сумма покупки с учетем скидки 

Для анализа был выбран период в 1 год, что является достаточным для подсчета динамики продаж по клиентам.

Если в момент совершения покупки касса находилась в оффлайн-режиме, то вместо номера карты фиксируется зашифрованная последовательность символов, что делает невозможным восстановление номера карты силами базы данных. Поэтому в нашем исследовании рассматриваются только те покупки, которые можно однозначно идентифицировать.

Также из анализа были исключены покупатели, сделавшие всего одну покупку, поскольку предложение персональной скидки для них не имеет смысла в контексте увеличения среднего чека.

## Выбор метрик
В бизнесе для описания распределения случайных величин используются различные метрики, которые помогают оценить характеристики данных и прогнозировать тренды. Некоторые из наиболее распространенных метрик включают:

- Среднее (mean): Это является наиболее простой и наиболее часто используемой метрикой. Оно показывает центральную тенденцию данных и равно сумме всех элементов в наборе данных, деленной на их количество.
- Медиана (median): Это значение, разделяющее данные на две равные части. Применяется в случае, когда распределение является симметричным или имеет выраженный скеw.
- Квартили: Квартили разделяют данные на четыре равных части, и каждый квартиль отображает определенный процент данных. Например, первый квартиль соответствует 25-му проценту, а третий квартиль - 75-му проценту данных.
- Децили: любое из девяти значений, которые делят отсортированные данные на десять равных частей, так что каждая часть представляет 1/10 выборки или генеральной совокупности.
- Перцентили: значение, которое заданная случайная величина не превышает с фиксированной вероятностью, заданной в процентах. Таким образом, N-й процентиль — это то же, что N-й квантиль.

## Подсчет метрик
Для подсчета величин был использован следующий код на PostgreSQL:
```
select card, 
	'2022-06-10'::date - max(d.datetime)::date as R,
	count(*) as F,
	sum(d.summ_with_disc) as M
from bonuscheques d 
where  substring(d.card from '^(\d+)(/.*)?$') is not null
group by 1
having count(*) > 1
```

Итого получилось:

Для распределения были подсчитаны все вышеописанные метрики с помощью PostgreSQL:
```
select metric,
	case 
		when metric = 'Среднее' then avg(R)
		when metric = 'Медиана' then percentile_disc(0.5) WITHIN GROUP (ORDER BY R)
		when metric = '1-й квартиль' then percentile_disc(0.25) WITHIN GROUP (ORDER BY R)
		when metric = '3-й квартиль' then percentile_disc(0.75) WITHIN GROUP (ORDER BY R)
		when metric = '33-й перцетиль' then percentile_disc(0.33) WITHIN GROUP (ORDER BY R)
		when metric = '67-й перцетиль' then percentile_disc(0.67) WITHIN GROUP (ORDER BY R)
		when metric = 'Максимум' then max(R)
		when metric = 'Минимум' then min(R)
	end as Recency,
	case 
		when metric = 'Среднее' then avg(F)
		when metric = 'Медиана' then percentile_disc(0.5) WITHIN GROUP (ORDER BY F)
		when metric = '1-й квартиль' then percentile_disc(0.25) WITHIN GROUP (ORDER BY F)
		when metric = '3-й квартиль' then percentile_disc(0.75) WITHIN GROUP (ORDER BY F)
		when metric = '33-й перцетиль' then percentile_disc(0.33) WITHIN GROUP (ORDER BY F)
		when metric = '67-й перцетиль' then percentile_disc(0.67) WITHIN GROUP (ORDER BY F)
		when metric = 'Максимум' then max(F)
		when metric = 'Минимум' then min(F)
	end as Frequency,
	case 
		when metric = 'Среднее' then avg(M)
		when metric = 'Медиана' then percentile_disc(0.5) WITHIN GROUP (ORDER BY M)
		when metric = '1-й квартиль' then percentile_disc(0.25) WITHIN GROUP (ORDER BY M)
		when metric = '3-й квартиль' then percentile_disc(0.75) WITHIN GROUP (ORDER BY M)
		when metric = '33-й перцетиль' then percentile_disc(0.33) WITHIN GROUP (ORDER BY M)
		when metric = '67-й перцетиль' then percentile_disc(0.67) WITHIN GROUP (ORDER BY M)
		when metric = 'Максимум' then max(M)
		when metric = 'Минимум' then min(M)
	end as Monetary
from data,
(values('Среднее'), ('Медиана'), ('1-й квартиль'), ('3-й квартиль'), ('33-й перцетиль'), ('67-й перцетиль'), ('Максимум'), ('Минимум')) as m(metric)
group by 1
```
